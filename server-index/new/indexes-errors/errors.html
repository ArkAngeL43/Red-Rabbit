0x000ERRR - Interacting with red rabbit on an error basis
This section of the script will primarily be talking about JUST errors that can happen inside of red rabbit, known bugs, and latest update on the errors and bug fixes this section wont be really split up much since there is no way to organize the errors other than the modules red rabbit will use like common errors with the requests module which will usually start an error message like so and so
> Windows Based Errors Red Rabbit has never been a windows supportive tool in fact it was brought up to steer as farrrrr away as possible from windows system support during development however some people still tried to run it on windows, resulting in well different errors. In red rabbit's source code it imports alot of custom made modules and some of those module's like the gopacket module need certian header files or may even need certian paths that are not supported in that code, when trying to run red rabbit on a windows based or windows system you get many errors such as files could not be installed properlly due to NULL input, or the module could not find the filepath specified, or even that the perl Makefile.pl file can not install itself in a directory or move the files to a new destination because that destination file simply does not exist. All of these errors are being worked on as the version 1.0 release, eventually red rabbit will have good and decent windows support, but for now it stays primary to arch or debian based linux distributions
> Errors when installing red rabbit there are a few issues you can get when installing red rabbit most of this being due to non proper install during module or third party module installs. A common one that seems to pop up even with currently testing red rabbit v 1.0 on linux has been insufficient and really annoying to fix. Currently the error you will get is during install of the gopacket third party library is that pcap.h is not a directory and when you use command
sudo apt-get install libpcap-dev
you get the error that the libpcap has a insufficient version and you may need to install a certian version based on its depends. This is a wacky error and one there is no way of fixing that is known, even after hours of trying to fix it there was no luck in actually trying to fix the issue. So in conclusion of this error it is very very annoying and simply will stop compiliation or instaklling of red rabbit any version really and if you can run wireshark and not red rabbit because of a simple pcap error your system is what would be assumed as broken
