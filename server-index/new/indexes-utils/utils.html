0x0004 Interacting with databases, SSH, Generators, regex, and file utilities
This section talks specifically about the utilities, etc, ssh, db functions and utilities including things like shells, SSH test utilities, beta utilities, host blockers and more this section will kinda knock out multiple sections in a row instead of splitting them up, if they are in any relation to each other, an example being the utils blocker will include all 3 parts of that in one section, to simplify it and make the process go by faster

this means that all hosts should not be accessible now utils blocker s does the same thing but with one host instead of a list given that our file usually can be annoying to work with, there is a function in red rabbit and a command named utils blocker fix which will reset the file back to normal based on templates. > utils verify | goes for any command that does verification There is not much to this, the reason i added this into the script was to make sure your hostnames, emails, and MAC's or other data would pass verification by red rabbit to be used in other attacks so if you wanted to make sure a hostname works or something like host:port would work in something like web recon or scanning you could use this the regex verification tools include verifying and checking that all of the following can be used in red rabbit without issues mac addresses email addresses IPv4 addresses IPv6 addresses Host:port formatted host addresses there is no output for this
> utils db | all commands that include database utilities This section here is specified to databases which are supported as the following : : PostGreSQL : MySQL : Mongo : Microsoft SQL there are a few commands set here and all of these are pretty much the same the first one in this is ping databasetype, ping will simply check if a database is alive on a given host, and that a port scanner did not give off a false positive or that it is not a honeypot and the service truly isnt up. The output will just be a true or false message, and the only way to use this is to enter the host / ip of the server. here is an example of using the ping utility to see if my local database is alive
[+] Setting: Pinging Host | 127.0.0.1 [+] Setting: Port | 5432 [+] Setting: Pinging host | 127.0.0.1:5432 [+] Information: Got a good ping on the host and dial, database is alive
the last command apart of the db section will be ` db auth databasetype` this will simply check if a verification in the server is correct, and if your configuration such as username and password and host are all correct this stands for simplicity really not for everyday usage this is also the reason it is set as an extra

Well if you want to you can do this yourself simply type utils gen JPG and you can use the lat and lon from really any data you find or output you have from any tool and generate an image

    > ssh | including all utils secure shell related > ssh check - The first command on list for this is ssh check this command simply checks to see if a SSH server on a given host is alive, simple and for this there is no intense or hyper output to go over > ssh transfer - second on the list is ssh transfer, this was ripped off from my old script and program named SSH-Sploit which used SCP to transfer files to a SSH hostname, simply just making the SSH posession process alot easier > ssh auth - third command on the SSH list is ssh auth this command will authenticate a ssh server based on your own configuration which requires a username and password > ssh auth key - fourth command on the SSH list is ssh auth key this command uses the ssh key directory to be authenticated with a username and password simple, and plain. > ssh auth config - fifth command on the SSH list is ssh auth config like many of the other config based options this will use a custom configuration files in YAML the path for this file is going to be in the /config/services/ssh path, if the file is not located or around then this file may be lost, if not please look in the same directory > ssh exec - sixth command on the SSH list is `ssh exec` this command allows you to sign onto a SSH server via password and username and capture the output of the command and report it back to the server > ssh dial config - seventh command on the SSH list is `ssh dial config` this command uses the same config file as set before but allows you to ping the SSH server / host and check to see if it is alive. if the server is alive the report would be a simple status message. > ssh check config - the eighth command on the SSH list is the `ssh check config` using the configuration file simply check if the SSH server is alive or not, if it is alive like above a simple stat message will keep alive. > ssh start shell - the final most useful command of all on the SSH listr will be `ssh start shell` this is a interactive SSH shell that will allow you to execute any command remotely this requires a private key and a username to be executed properly.
