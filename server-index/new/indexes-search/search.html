0x0001.5 interacting with vulnerable services and servers
This next section basically shows off alot of the scanners, and extra utilities for web recon, web scanning, system scanning etc
> search admin this command will read a wordlist of admin paths such as a custom one, and take a URL to search all admin paths of that URL that came back true. Note this goes based off of a response code and is truly not the best to use since false positives may exist as a security measure on the host or server. example output
Enter path to admin panel payload list or a payload list>examples2/admin.txt ------------------------------------------------------------- RR6: Web Module: Test -> [ 1 ] Has come back false ------------------------------------------------------------- RR6: Web Module: Test -> [ 2 ] Has come back false ------------------------------------------------------------- RR6: Web Module: Test -> [ 3 ] Has come back false -------------------------------------------------------------
> search sqli this command will search for SQLI vulnerabilities in a URL or host based on the error syntax that it gets from the SQL server on the backend, if the error is any of the following in the list of code, then it will say positive and keep testing for more vulnerable errors. example output
RR6> SQLI Stat: Could not inject payload -> id=1 Payload failed, testing next RR6> SQLI Stat: Could not inject payload -> id=1 Payload failed, testing next RR6> SQLI Stat: Could not inject payload -> id=1 Payload failed, testing next RR6> SQLI Stat: Could not inject payload -> id=1 Payload failed, testing next RR6> SQLI Stat: Could not inject payload -> '-' Payload failed, testing next RR6> SQLI Stat: Could not inject payload -> '-' Payload failed, testing next RR6> SQLI Stat: Could not inject payload -> '-' Payload failed, testing next RR6> SQLI Stat: Could not inject payload -> '-' Payload failed, testing next RR6> SQLI Stat: Could not inject payload -> ' ' Payload failed, testing next RR6> SQLI Stat: Could not inject payload -> ' ' Payload failed, testing next
> search lfi This command is a simple LFI scanner, there really is no output or information to give on this utility other than that it uses NULL entries to detect RFI/LFI and is quite speedy.
> search sig search sig will attempt to find the filetype of an unknown file based on hex signatures and binary signatures. Sometimes this tool can glitch if the byte code in the file is super large, given it itterates over a list of hex values and byte values to test which one matches up to the files header example output

> search geo this command takes the input or path using the flag -i of an image the image MUST be a JPG image, and yes there are byte checkers to ensure that the file is JPG and not just named JPG to hide it, but it will take the` image and search for the geolocation / GPS tags and attempt to generate a map link of google, generate a GPS image then pass onto output the lat and lon of the image GPS cords, this again is outputted into an image, but it depends on if the user allows GPS perms on their device when taking images. there is no example output here for a reason, note there are some issues with this.
