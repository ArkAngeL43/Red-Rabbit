> inject jpg, webp, bmp All of these commands inject jpg inject webp inject bmp all do the same exact thing, inject a payload into an image is the basis of all four of these commands
> inject png This command is apart of the stegonography toolkit and section that red rabbit holds, however this module is very very much different than any of the other stegonography modules and toolkits that red rabbit has in its selection. This image injection module is still being upgraded since it was built on a module written by the authors of the black hat go book, credits to them for such an amazing oppurtinity to inspire my team to look into stegonography and even modify the toolkit they taught us to build that i implimented into the red rabbit project. Currently this module can do a few things such as- -> Inject a simple payload into a PNG image -> Inject a encoded payload into a PNG image -> decode a payload in a PNG image -> Get meta and image information on the PNG image This toolkit also inspired some other things such as stegonography with archives which we will discuss next inject png is just one command however it has a few flags that can be used with it, in order to properly use this command and sucessfully inject your payload into the PNG image you must do a few things like specifying flags, for the most common usage of this script you can run the following Running script without being compiled | sudo go run main.go -i="path/to/image/image.png" -o="output_of_injected_image.png" --payload="some_payloas-02394890234890238904238904238904809342" Running script in binary form sudo ./main -i="path/to/image/image.png" -o="output_of_injected_image.png" --payload="some_payloas-02394890234890238904238904238904809342" example output below of a simple injection being made to a PNG image
