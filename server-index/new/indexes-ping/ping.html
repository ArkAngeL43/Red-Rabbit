0x0012 interacting with networks via ping
Section 0x0012 will be talking about all 4 ping commands that can be used for host recon, simply all these do is check which host is alive using different methods, however the one in here will need extra descriptions the reason there are different packets that can be used such as UDP, TCP, ICMP is because some hosts respond or are reported alive fatster than others when using different packet methods NOTE: when running any ping command before you run red rabbit you must specify -z or --CIDR or the defualt scan will be [+] RR6> LOG: Settings: CONF -> 192.168.1.8/24
> ping icmp Ping icmp is a command that will send out ICMP based packets and ping all hosts on the network trying to find out which host is alive, for every host alive a table row will be added looking like the following after each report an example of using 10.0.0.1/24 as a range from a reported beta tester

> ping arp this one is a bit different from all ping commands. The reason being is it has a few features, one being it sends its own custom packets and waits for replys, another being it starts on individual threads and captures responses through each channel and reports it. The feature that makes this one different from every other feature in this list isnt the fact that it doesnt stop until you say it can stop but it will trace the OUI of each mac it finds on each IP take the following example
