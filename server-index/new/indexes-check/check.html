0x0006 Number tracking, username trackers, and other OSINT utilities
Seciton 0x0006 will be talking about cloud checking utilities, phone number databases and proton OSINT scripts.
> check cpanel vuln this command will simply detect the system you are on attempt to find the cpanel admin path, and then finally check to see if the current version you have is vulnerable if it is it will link you to a perl file that you can run to upgrade and repair your current version of cpanel
> check proton ip check if an IP address belongs to the proton mail / proton VPN database, if the IP is apart of the database you will get the following message during runtime
Enter a IP>> 1.1.1.1 ---->>> RR6: IP [1.1.1.1], is NOT apart of the proton mail VPN...
> check proton email like check proton IP this will simply ask you for an email, pull the email list then find out and search to see if it belongs to the proton mail email service. when you run the tool and the tool found the email you will get the following message.
Enter an email>> example_email@protonmail.com found it!

> check cloudfront ip this command goes through the cloudfront global and regional edge IP list and tests if your IP is a match for one in the list you will get the following message example output
(Cloud Check) IP < 1.1.1.1 > is NOT apart of the cloudfront (CLOUDFRONT_GLOBAL_IP_LIST) (Cloud Check) IP < 1.1.1.1 > is NOT apart of the cloudfront (CLOUDFRONT_REGIONAL_EDGE_IP_LIST)
> check mcafe ip simple command, this one like all cloud utils cloud related will check the json database that is stored in the file, and attempt to match an IP you entered with the IP addresses in the list if your IP came back false and did not match no output will be found, if it did you will get a simple message saying the IP is apart of the mcafe list.
> check aws ip like all cloud checking utilities this one will take an ip address and check the AWS IP list and JSON db to see if the IP matched the database. Note that if the aws json file is not in the directory that it needs to search, it will make a request to grab the json file and return to the console, you will need to run the command again to check if your IP matches the AWS IP LIST
> check azure ip Command will check the IP address to see if it is apart of the microsoft azure cloud IP database, if it is true it will return a simple message if else then it wont return anything.
> check myip Command will grab your IP address no output is needed to be shown literally all it does is show your public IP
> check number the number section is a bit wacky so here i am to explain it the way this works is the function accepts a number hash value as a input like +381-##-###-####, and it will attempt to find and match the hash to a section in the database, then it will pull the location of that number or base location, this does not output the direct location it will mostly be a state or country example output

> check number be this is similar to the number section however unlike the check number or defualt international entry, this is specified to numbers that are assigned or collected or ran in the beligum country or area, this like the previous command takes a number hash based value like the following (+32(2)###-##-##) this version of the scanner will grab everything about the number, the area of the number, and the aproximate information about the number hash, there is soon to be way more added to this in the further future such as time zones, time current, weather, google maps links, image maps, and more. example output

> check number us this one like all number commands will parse through a database of information specified to the numbers in the US only. this also operates based on a hash like input but instead of adding ### as a format you can just add the first three numbers of the phone number such as the area / dial code. example output

> trace number us this is unlike most of the phone number / phone hash checkers in red rabbit, this one uses a few phone API's to grab information on the phone number that may be in use, however this tool will need the entire number. note: since this is a bash file the tool might not capture the output that you need, or it may not capture any at all, this is probobly due to a internal error in the OS or the script that pops up or is detected as NIL or an ignored error by red rabbit.
