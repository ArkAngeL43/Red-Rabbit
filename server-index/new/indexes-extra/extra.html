A breif intro as to how the red rabbit terminal works

I did not want to go over this or bother making this a section in other command like sections so why not do it here. Red rabbit works alot like metasploit in terms of running options, it has a module, each module has its own utility, say the module search right. The search module stays static but its subroutines dont `search mx` changes at mx `search head` changes at head. Take the following tree as an example of how modules work
| - engine |> engine ogoogle |> engine otwitter |> engine ofacebook |> engine olinkedin |> engine cve |> engine tcve |> engine shodanh
so do note, all options that you input whatever the first thing you type to run that module is going to be the module name, a way to shorten the giant ass command list is to type `help module_name` example as `help inject` will output all information and commands under the module inject in a organized and fit manner

Proxy downloading

> fetch proxy4, fetch proxy5 both of these commands do the same thing. what they do is download proxies and save them to a output file you specified wether it was from using the -o flag or by manually typing it when the program asks for it. however the difference is proxy4 will fetch all proxies it can for SOCKS4, proxy5 will fetch all proxies for SOCKS5 > fetch http/https/httpssl/httpsssl like the one above all of these are similar, specify an output file and it will download and save proxies to that given file. fetch http/https commands will fetch all proxies relating to a http/https like address, there are options to also use SSL for the proxy grabbing list. fetch htpssl and httpsssl will grab proxies of either http or https and make sure they are specified to just SSL, anything without SSL will not be added to the list.
